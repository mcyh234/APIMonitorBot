###2### plan
API 状态监视器 Bot 实施计划
Summary
从空项目实现一个本机单服务：FastAPI 后端 + React WebUI + SQLite。后端负责 API 可用性巡检、OneBot v11 通知发送、OneBot 消息接收、管理员命令、多轮添加配置、权限控制和加密存储。
OneBot 接入分两层：
发消息：HTTP API 调用 send_group_msg / send_private_msg，支持 Authorization: Bearer <access_token>。
收消息：优先支持 WebSocket 连接 OneBot 事件流，连接时携带 access token；同时保留 HTTP webhook 入口作为备用传输。
Key Changes
后端栈：FastAPI、Uvicorn、SQLite、SQLModel/SQLAlchemy、Alembic、httpx、APScheduler、websockets、cryptography。
前端栈：React + Vite，构建后由 FastAPI 静态托管。
本地配置：.env 包含 ONEBOT_API_BASE_URL、ONEBOT_WS_URL、ONEBOT_ACCESS_TOKEN、SECRET_MASTER_KEY、APP_HOST=127.0.0.1、APP_PORT。
密钥策略：APIKey、OneBot token 使用环境主密钥加密后存 SQLite；Web API 不返回明文密钥。
默认管理员：启动时自动创建 QQ 管理员 1234567890，后续支持 WebUI 管理多个管理员。
Backend Behavior
建立 OneBot 客户端模块：HTTP sender 负责群聊/私聊通知。
WebSocket receiver 负责接收私聊和群聊消息事件，断线后自动重连。
WebSocket 握手携带 Authorization: Bearer <token>；必要时兼容 query token 配置。

命令系统：/list：仅管理员，列出配置名。
/remove <apiname>：仅管理员，删除检测项。
/addapi：仅管理员，进入多轮添加流程；APIKey 收集强制私聊完成。
/check <apiname>：群聊只允许查询本群绑定的 API；私聊仅管理员；每个 QQ 用户 5 分钟限一次。

/addapi 流程：配置名 -> 报告目标 G群号 / PQQ号 -> BaseURL -> APIKey -> 模型名。
保存前立即调用 OpenAI 兼容 chat/completions 发送 hi 验证。
验证成功后入库并启用巡检。

Monitoring Behavior
每 1 分钟检查所有启用 API。
探测请求：POST {base_url}/chat/completions，使用对应 APIKey、模型名和 hi 消息。
可用判定：HTTP 2xx 且返回 assistant 内容；401/403/429/5xx、超时、网络错误、JSON 异常都算不可用。
首次失败后立即二次确认；二次仍失败才进入不可用状态。
通知策略：首次确认不可用：立即发送“当前出现业务中断”。
10 分钟内不重复发送不可用通知。
10 次巡检后仍不可用：发送“10分钟仍未恢复业务”，之后静默。
故障后连续 2 次恢复成功：发送“当前服务恢复可用”。

今日可用性按 Asia/Shanghai 自然日统计，只统计定时巡检结果。
WebUI
首页显示所有 API 配置、当前状态、今日可用性、最近错误码、最近检查时间。
支持新增、删除、启停、立即检查 API 配置。
支持查看单个配置的巡检历史。
支持管理员 QQ 列表管理。
第一版 WebUI 仅监听 127.0.0.1，免登录；若以后开放局域网/公网，再补登录鉴权。
Test Plan
单元测试：BaseURL 拼接、API 探测解析、密钥加解密、命令解析、权限判断、限流判断。
状态机测试：首次故障、10 分钟仍未恢复、连续 2 次恢复。
OneBot 测试：mock HTTP 发送接口、mock WebSocket 消息事件、验证 access token header。
集成测试：模拟 /addapi 多轮对话、配置入库、定时巡检、告警通知。
前端测试：构建通过，页面能完成配置 CRUD、手动检查、历史查看和管理员维护。
Assumptions
第一版默认使用 OneBot WebSocket 收消息，HTTP POST webhook 作为备用。
第一版不接 Telegram，但通知发送层预留 Notifier 抽象，后续可增加 Telegram 实现。
WebUI 默认只本机访问，所以暂不做账号登录。
群成员身份校验以 OneBot 能提供的信息为准；无法确认时只允许管理员操作。
